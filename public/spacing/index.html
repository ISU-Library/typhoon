Spacing

Typhoon spacing guide

Overview

Typhoon uses pre-set spacing design tokens and utility classes to streamling design and speed up development. Both tokens and utility classes use a descriptive naming convition instead of numeric values to improve flexibility.

Spacing Utility Classes

There are utility classes for both padding and margins in Typhoon. Each follows the same structure of the library prefix, the type of spacing and which part of the element, and the size. For example lib-pt_med provides padding (p) to the top (t) of the element. The amount of spacing is the medium spacing token. If you wanted to do margins, you would use lib-mt_med instead. Padding and margin have the same class except margin, which has an lib-m_auto, which set the x-axis margins to auto for horizontal centering.

Class Example Display Description
lib-m_med
Adds margins on all side of an element
lib-mx_med
Adds margins on the x-axis of an element
lib-my_med
Adds margins on the y-axis of an element
lib-mt_med
Adds margins on the top side of an element
lib-mr_med
Adds margins on the right side of an element
lib-mb_med
Adds margins on the bottom side of an element
lib-ml_med
Adds margins on the left side of an element
lib-m_auto
Sets x-axis margins to auto to horizontally center element

Spacing Sizes

The spacing modifiers are preset to improve the design process. The med from the abouve exmaple can be replaced with any of the below values.
